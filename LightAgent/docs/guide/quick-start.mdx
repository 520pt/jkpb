title: One-click Install
description: One-click install and manage LightAgent with scripts
---

The project provides scripts for one-click install, configuration, startup, and management. Script-based deployment is recommended for quick setup.

Supports Linux, macOS, and Windows. Requires Python 3.7-3.13 (3.9 recommended).

## Install Command

<Tabs>
  <Tab title="Linux / macOS">
    ```bash
    bash <(curl -fsSL https://raw.githubusercontent.com/yideng966/LightAgent/master/run.sh)
    ```
  </Tab>
  <Tab title="Windows (PowerShell)">
    ```powershell
    irm https://raw.githubusercontent.com/yideng966/LightAgent/master/scripts/run.ps1 | iex
    ```
  </Tab>
</Tabs>

The script automatically performs these steps:

1. Check Python environment (requires Python 3.7+)
2. Install required tools (git, curl, etc.)
3. Clone project to `~/LightAgent`
4. Install Python dependencies and LightAgent CLI
5. Guided configuration for AI model and channel
6. Start service

By default, the Web console starts after installation. Access `http://localhost:9899` to begin chatting.

<Note>
  **Deploying on a server?** By default `web_host` only listens on `127.0.0.1` (local access only). Set `web_host` to `0.0.0.0` in `config.json` to make the console reachable from outside, and set `web_password` to protect it. Don't forget to open port `9899` in your firewall or security group — ideally restricted to specific IPs.
</Note>

## Management Commands

After installation, use the `lightagent` command to manage the service:

| Command | Description |
| --- | --- |
| `lightagent start` | Start service |
| `lightagent stop` | Stop service |
| `lightagent restart` | Restart service |
| `lightagent status` | Check run status |
| `lightagent logs` | View real-time logs |
| `lightagent update` | Update code and restart |
| `lightagent install-browser` | Install browser tool dependencies |

See the [Commands documentation](/cli/index) for more details.

<Note>
  If the `lightagent` command is not available, you can use `./run.sh <command>` (Linux/macOS) or `.\scripts\run.ps1 <command>` (Windows) as a fallback. Both are functionally equivalent.
</Note>
